Blockchain and Cryptocurrencies


A blockchain is a distributed and immutable ledger that can be adapted to record many different types of transactions, typically these are maintained across several computers that are linked together in a peer-to-peer network using high level cryptography. As the namesake suggests, information is stored together in batches called blocks that are linked chronologically together to form 'chains'. Cryptocurrencies such as Bitcoin utilise blockchain technology to record transactions, and are a form of digital currency that is decentralised and is generated by the solving of complicated mathematical problems by computers.

There are three main types of blockchains:

1. Public (permission-less) blockchains allow anyone to register as a node in the network and each user has a copy of the blockchain. It also allows users to examine a complete history of transaction details.

2. Private (permissioned) blockchains require consent to be added to the network. Nodes are therefore trusted and there is a higher degree of privacy and control over who can see transaction information.

3. Hybrid blockchains utilise the best elements of private and public blockchain solutions, they are flexible in how they can be implements and about which parts of the transactions are visible and private to users.

Each block contains batches of previous transactions that occurred before it was created and a cryptographic hash of itself and of the block preceding it, thereby verifying the integrity of the chain. Attempts to tamper with the metadata will render the hash on the following block invalid. If a single character or value is altered it completely changes the hash output. One would need to simultaneously modify all of the blocks back to the genesis block in order to tamper with this data.
The simplified process for determining which transactions comprise the next block are as follows;

1. Client makes a transaction and it's added to the pool of pending transactions.
2. Nodes detect changes and validate all other transactions that have occurred on the blockchain so far are legitimate and unchanged.
3. 'Emulation of a block' and the proof of work algorithm is initiated – nodes on the network work together to solve a cryptographic hash of the block by guessing random values.
4. Once solved, the block generates a unique hash output and unlocks the position of the next block. It broadcasts to all other nodes on the network and this is used in a proof of stake test to purchase entry into the blockchain.

Using Bitcoin as an example of a public blockchain, whichever node on the network is first to solve the cryptographic puzzle is awarded a single Bitcoin, this process is commonly referred to as 'mining'. With the aggregate computing power of millions of computers worldwide this is estimated to occur once every ten minutes, ensuring a steady supply of Bitcoin is circulating without devaluing the currency. In order to send Bitcoin to another user, the system must first reference previous transactions (or inputs) that demonstrate the account has the sufficient balance to afford the transaction. If the transaction has been used once as an input it is regarded as being used or spent and cannot be used as a future input, other nodes verify this by checking every transaction ever made to confirm it has not been used as an input, and thus this method solves the 'double spend' problem. Some of the state of the art algorithms being utilised by cryptocurrencies include the elliptic curve digital signature algorithm (ECDSA) and mathematical trapdoor functions which help to ensure that funds cannot be accessed by individuals other than the owner.

The simplified process for authenticating a transaction is as follows (usage of both of the algorithms mentioned above);
1. Private key creates a public key that other users verify - this process is unidirectional (difficult to find the inverse). And the digital signature (hash) acts as the intermediary.
2. A digital signature is generated from a combination of both the transaction message and private key.
3. Other nodes on the network verify that this signature corresponds with the public key, and therefore can verify that it corresponds with the private key, without actually needing to see it. Because the signature is dependent on the transaction message it will be different each time and cannot be reused in a different transaction and cannot be modified/tampered without invalidating the signature.

Future and potential impact on daily life

Blockchain and cryptocurrencies are quite exciting developments and I'm of the perspective that these technologies are still very much in their infancy. There are many instances of these technologies being applied today in the real world, and seemingly limitless possibilities for their utilisation in the coming years. The future will depend on how governments, legislators, businesses and other major players choose to respond and adapt to these imminent shifts in technology. Practically any industry one can think of is scheduled for large scale transformations in regards to the way business is conducted. Financial industry; The most obvious sector being the financial industry which is set to be challenged on just about every dimension of how it operates. An example is a service such as transferring money internationally, which is not only lengthy, often taking several days for settlement to resolve, but is also accompanied by a hefty transaction fee. Cryptocurrencies threaten the existence of such a service as it occurs in near real-time, due to payment and settlement occurring in the same step in the process, as well as for a fraction of the transaction price.

Smart contracts;
Various cryptocurrencies have begun implementing smart contracts, a computer protocol which digitally facilitates the negotiation and execution of a contract. The contract can be fine tuned so that the transaction will only occur when the conditions are a perfect match, and the specifications of this procedure are then stored on the blockchain or distributed ledger. This is advantageous as this process can be automated, there are additional levels of security and accuracy, and reduced costs without the need for intermediaries. It stands to reason that both parties involved in the transaction benefit greatly.

Non-fungible tokens (NFT);
Most cryptographic tokens are interchangeable and comprised of parts that are indistinguishable from one another (fungible), however an NFT has the capacity to store unique and far more complicated sets of information. Today NFTs are primarily being utilised as a token on the Ethereum blockchain, but moving forward in time they may prove to be well suited for the storage of different types of sensitive information such as government documentation (driver's licenses, records of immutable land title and assets) and for verifying the legitimacy of tangible goods.

Medical record-keeping;
Blockchain technology can be integrated with pre-existing electronic health records technology for ease of access to up-to-date health information about a particular individual. A private blockchain can be set up to store this information and render it unalterable and only accessible by permitted users. This may also have the added benefit of enhancing individual privacy by removing the centralised host from the equation.

Decentralised cloud storage;
 Blockchain can be utilised here to create a decentralised storage hub that isn't owned by any one individual company. This would be particularly effective at bolstering security of information, because a key factor at play relating to data leaks is a large centralised information storage that becomes compromised. Segmenting data in this way would thoroughly alleviate this potential security weakness. It may also serve to enhance privacy and circumvent some of the restrictions imposed by existing cloud services.

Machine Learning

Machine learning is the task of allowing a computer to follow an algorithm to create an outcome. Machine learning is used in a wide range of uses, such as clerical work, customer service, and fraud detection. Many industries have also started using machine learning to allow for better medical diagnosis and personalised healthcare. Machine learning is an umbrella term for multiple ways computers can be used to produce something from algorithms. In this short time, I will present how machine learning works, the impact machine learning brings, and how machine learning personally impacts me.


Machine learning, while not a new technology introduced to the world, it is a form of technology that we are now seeing being used in many different applications across a wide range of industries. From simple mobile phone applications to the use in fraud detection. Machine learning is an umbrella title that encompasses many different ways of allowing computers to perform repetitive tasks and provide outputs. To educate how machine learning works is quite simple. Say you have a job which requires you to say whether or not the face in front of you is of a man called John. Now for a human, you can quickly know if the face is of John or not. However, for a computer, how could it do it? Well, one way could be to find the measurements of John's face. These measurements of Johns face become apart of the data; machine learning would then need other pictures of random people and even more of John to allow the machine learning to decide if the image is of John or not. After seeing all this data, the algorithm should be able to say if the image that is given is of John or not. This is only just one of the ways machine learning is being used to do the work that once a human had to do. Other forms of machine learning are trying to find objects in an image. Google, is one company who is tackling this issue of getting machine learning to figure out what object is in an image. One way they are teaching machine learning to figure out what object is what is by enlisting almost everyone who uses the internet. When signing up for a new user account or submitting a form to a website sometimes you may be asked to find the traffic light in a set of nine images before you can press the submit button. By selecting the right pictures, Google can train the algorithm at a much faster rate than a team doing all the work in house.


Machine learning has already created a significant impact on the world, and everyday people are advancing the abilities of computers performing these complex tasks that would take the human brain a plethora of time to complete. The rise of machine learning will make a more substantial impact and impression on the world, as, without machine learning, the world would not be able to grow and advance in such fields as autonomous driving or even medical diagnosis. However, machine learning also brings the question "Who or what will be replaced?". With the advancement of machine learning, the world has already seen several reports authored stating roles such as clerical work and manufacturing work could be placed on to the chopping board as computers are better at performing those repetitive tasks. However, with the introduction of machine learning, many industries and companies believe that machine learning will introduce more jobs in the market. Companies such as Amazon have pledged to upskill their employees to allow them to become familiar and better suited to work within the machine learning filed. As well as companies pledging to educate their employees, we are now seeing many universities introducing courses focused around machine learning to allow future students to immerse themselves into this dynamic field.


Machine learning already does and will continue to play a significant role in my life. Currently, machine learning is almost featured in the majority of the mobile applications I use on my phone on a daily basis. Those mobile phone applications apply machine learning differently and use their capabilities to perform tasks that would appear not to be aided by machine learning. Most nights I'll unwind after a busy day by watching some TV. However, in this day and age I'm not turning on free to air television, I'm opening one of the many on-demand video streaming services available today. These services invest large amounts of time and money into machine learning to display content to myself which a computer and an algorithm, and a vastness of data about myself, which some of I may not even know, to figure out what I will like to watch. In actions like this, when educated on topics such as machine learning, it is easy to discover that companies have deployed this technology. Now, machine learning is appearing in places where machine learning would seem like such a big introduction for such a small product. For example, I use another mobile phone application to aid my efforts when working out at the gym. Not so long ago, most applications for the active person would try to create a "personalised" workout plan for you. Still, the meaning of personalised did not actually mean tailored down to a tee, but instead slightly adapted after receiving minimal data from the user regarding, weight and height. Now, machine learning has been introduced to process the data given from my Apple Watch or my smart scales to allow for a truly personalised workout plan suited for me made by a machine and an algorithm.Â

Without NLP in Chatterbots, you would be left with a very rough and somewhat useless version of an outdated Chatterbot. Which would not be able to accurately predict with the user is asking for and could end being vastly different then what they were asking for.
